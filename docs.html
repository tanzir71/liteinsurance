Deploy to shared hosting

Upload liteinsurance.php into public_html/, ensure PHP 8+ and PDO SQLite are enabled, then visit /liteinsurance.php. The first registered user becomes the admin. Settings includes a Setup doctor for PHP, SQLite, paths, upload limits, cron token, and webroot storage checks.

public_html/
  liteinsurance.php
  .env             # optional

Visit: https://YOURDOMAIN/liteinsurance.php
Register first user -> Settings -> Setup doctor -> Import -> Preview -> Confirm import
DOC / 02

Run locally

The app has no build step. Use PHP's built-in server from the repository root.

php -S 127.0.0.1:8000
open http://127.0.0.1:8000/liteinsurance.php

DOC / 05

Risk rules

Rules are priority-ordered JSON definitions. A matching rule can set risk_tier, adjust score, and add a tag. Supported leaf operators include =, !=, >, <, >=, <=, in, contains, and regex.

{
  "logic": "AND",
  "conditions": [
    { "field": "age", "op": ">=", "value": 60 },
    { "field": "tenure_months", "op": "<", "value": 12 }
  ],
  "actions": {
    "set_risk_tier": "High",
    "score_delta": 20,
    "add_tag": "senior_short_tenure"
  }
}
DOC / 06

Segments and exports

Segments combine selected rule IDs with a validated WHERE-style filter. Allowed fields include core fields plus custom fields using custom.field_key. CSV export is protected by a session token and appends enabled custom fields.

risk_tier = 'High' AND ltv_estimate > 7000
custom.renewal_probability >= 0.7 AND confidence_score >= 80

Security model

Web controls

CSRF on POST, session hardening, idle timeout, rate limits, and security headers.

Data controls

PII is masked in list views. Uploads are randomized and blocked from direct browsing when possible.

mask_name
