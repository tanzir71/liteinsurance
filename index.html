FAQ / 05

Questions an evaluator will ask before uploading a real CSV.

Does LiteInsurance send policyholder data to a cloud service?

No. The production app is a PHP file backed by SQLite on your server. Imported CSV data stays in your hosting account unless you export it.

What are the server requirements?

PHP 8 or newer with PDO SQLite enabled. The default setup is designed for shared hosting and cPanel-style environments.

Can I try it before deploying PHP?

Yes. The no-install demo starts with a deterministic 200-policy dataset, then lets you replace it with a local policyholder CSV and edit risk rules JSON for dashboards, segments, and campaign math.

Is this a full agency management system?

No. LiteInsurance focuses on one slice: importing policyholder exports, cleaning them, scoring LTV and risk, segmenting, and simulating outreach.

What happens with missing CSV fields?

Mapped numeric fields use mean imputation; mapped categorical fields use mode or defaults. Unmapped columns are retained as custom fields, and LTV/risk scoring stays blank or Unknown until the required scoring inputs are mapped.

Can my CSV have custom fields?

Yes. The importer uses Record ID and optional Display name mapping, while custom CSV fields preserved locally can be viewed, exported, and referenced from rules or segments with custom.field_key.

How do risk rules work?

Rules are JSON definitions with conditions and actions. Matching rules can set a risk tier, adjust confidence, and add tags for segment logic.
